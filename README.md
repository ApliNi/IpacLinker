**IpacLinker**
是基于 [WebRTC](https://en.wikipedia.org/wiki/WebRTC) 的点对点网络连接工具.
可用于 Minecraft 联机和开服, 或者远程桌面 / 网络驱动器等应用的异地连接.

### 问题
- 连接是否加密: 是的, 我们使用的 WebRTC 库根据标准 (RFC 8826), 所有连接都会进行加密. 但服务端模式公开的端口需要自行处理加密问题.

### 使用方式

```
./IpacLinker --key value
```

#### 服务端配置
> 假设: 服务器中部署了 Minecraft 在 25565 端口, 同时还有一个 Web 服务在 8123 端口

示例配置: `./IpacLinker --server 0.0.0.0:10001 --token 0000 --ports 25565,8123`

- `--server 0.0.0.0:10001` - 启动服务器模式, 监听端口
- `--token 0000` - 设置 token
- `--ports 25565,8123` - 设置允许访问的端口白名单

#### 客户端配置
> 假设: 客户端通过 127.0.0.2:25565 连接服务器, 并可以在浏览器访问 127.0.0.2:8123

示例配置: `./IpacLinker --client ws://0.0.0.0:10001/ice --token 0000 --ports 25565,8123 --local 127.0.0.2`

- `--client ws://1.2.3.4:10001/ice` - 启动客户端模式, 连接服务端
- `--token 0000` - 要求与服务端 token 一致才能使用
- `--ports 25565,8123` - 映射哪些端口
- `--local 127.0.0.2` - 映射到哪个地址

#### 虚拟端口
这个功能用于隐藏真实端口和自定义监听端口:
- 服务端: `--ports 25565:25566` 表示实际连接 25565 端口, 但客户端必须通过 `--ports 25566` 才能连接
- 客户端: `--ports 25566:25565` 表示连接服务端的 25566, 但本地监听 25567 端口
- 也可以同时配置多个端口: `--ports 25565:25566,8080:80,5244`
